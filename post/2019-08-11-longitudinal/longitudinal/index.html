Lidando com dados longitudinais e pareamento de dados

Last updated on Aug 11, 2019 6 min read Estatística, Pareamento

Quando falamos em bases de dados, existem basicamente dois grandes tipos de dados: Transversais e Longitudinais.

Dados transversais (também chamados de cross-section) são o tipo de dado mais comum encontrado em qualquer área de estudo. Nesse tipo de base, existe uma observação para cada “indivíduo”. Como uma fotografia, ou um recorte, da população a ser estudada, um dado transversal tenta captar as variações que ocorrem na população através de uma imagem estática.

Um grande exemplo de dado transversal é a base de dados starwars do pacote dplyr:

Essa base mostra uma série de dados (altura, peso, cor de cabelo, pele e olhos) de diversos personagens da franquia Star Wars. Cada personagem aparece apenas uma vez (com exceção de Anakin Skywalker/Darth Vader, que aparece como dois personagens, mas divago).

O outro tipo de dado é o “longitudinal”. Nesse tipo de estudo, os dados são coletados repetidamente para ao mesmo indivíduo. Olhando para a base do Star Wars, se tivessemos dados da vida dos personagens, ela se tornaria longitudinal. Por exemplo, teríamos dados do Luke Skywalker com seu peso e altura em diversas fases da sua vida.

Uma base longitudinal, como você deve imaginar, é muito mais complicada de se obter, mas permite análises também mais complexas. É possível determinar tipos de crescimento, entender comportamentos antes/depois de certos acontecimentos (resposta a um remédio, sobrevida depois de uma cirurgia, medições de variáveis climáticas ao redor do globo). Conseguir uma base longitudinal leva tempo e custa caro, mas existem determinadas aplicações que necessitam desse tipo de dado.

Dentro da área florestal, existe um uso específico que exige dados longitudinais: os modelos de crescimento florestal. Para que seja possível entender os padrões de crescimento de um povoamento florestal, é preciso que sejam coletadas amostras de um determinado conjunto de árvores repetidamente ao longo do tempo.

Dessa forma, ficamos com uma base de dados como a abaixo:

Nesse caso, existem medições da mesma parcela amostral ao longo de 3 anos. Com esse tipo de dado é possível construir um modelo de crescimento e produção para aquele tipo de floresta.

Para que seja possível construir um modelo desse tipo, existe um passo extra necessário, é necessário organizar pares de observações, em linha, de forma a parear estas bases de forma a conseguir ajustar um modelo do tipo \(Vol_2 \sim 1 + I_2 + Vol_1 + I_1\), ou seja, um Volume no tempo 2 em função da idade no tempo 2, volume no tempo 1 e idade no tempo 1.

Um exemplo desse tipo de modelo é o de Lundqvist-Korf, em que uma de suas equações diferenciais pode ser escrita do seguinte modo:

\(Vol_2 = a*(Vol_1/a)^(I_1/I_2)^c\)

Para que seja possível ter uma base pronta para esse ajuste, é necessário que seja feito o pareamento dos dados, ou seja, cada idade deve ser combinada às demais. É basicamente uma combinatória com algumas regras:

  • Devem ser pareadas apenas as mesmas medições das parcelas;
  • A idade 2 deve ser maior que a idade 1;

Para fazer essa combinatória, elaborei uma função de pareamento, apresentada abaixo e aplicada aos dados do nosso exemplo. Apesar de ter pensado ela para dados florestais, a mesma pode ser usada com poucas modificações para quaisquer origem de dados.

A função terá 3 argumentos: - table: o data frame a ser pareado - id: o identificador único de sua parcela amostral - idade: sua variável indicando o tempo de medição. Pode ser idade, ano, mês… a coluna que indica QUANDO a medição foi feita, em relação ao \(t_0\).

pacman::p_load(dplyr)

pareamento <- function(table, id, idade){
  
  expand.grid.df <- function(...) Reduce(function(...) merge(..., by=NULL), list(...)) #Cria Função combinatória
  
  id_ <- sym(ensym(id)) #transforma inputs em strings e depois em objetos
  idade1 <- sym(str_c(ensym(idade), ".x"))
  idade2 <- sym(str_c(ensym(idade), ".y"))
  
  par_unique <- unique(table %>%
                         pull(!!id_))
  
  output <- vector("list", length(par_unique)) # cria output de listas vazio
  
  p <- progress_estimated(length(par_unique)) # Cria barra de progresso
  
  for (i in seq_along(par_unique)) {
    
    a <- table %>% filter(!!id_ == par_unique[i])  #filtra parcela i
    b <- expand.grid.df(a,a) #combinatória de parcela amostral
    c <- b %>% filter(!!idade2 > !!idade1) #separa i2 > i1
    output[[i]] <- c # cria lista i com output 
    
    p$tick()$print() #print barra de progresso
  }
  bind_rows(output) #combina output em um único data frame
}

#Aplicando a função a nosso exemplo
parcela %>% 
  pareamento(id = parcela_amostral, idade = idade)
